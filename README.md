La aplicación considera el uso de dos tipos de usuario, mesero y barman.
El mesero tiene acceso a la lista de tragos que se encuentran listo para servir en 
barra (una vez servidos puede sacarlos de la lista)y además a la sección en la 
cual puede tomar los pedidos, editándolos y confirmándolos.
El barman tiene acceso a la lista de tragos confirmados por los meseros, es decir,
los que debe servir (una vez los sirve los marca como listos para que esten en la
seccion de pendientes del mesero) además es el único usuario capaz de crear nuevos usuarios.
Esta aplicación esta ideada para un bar relativamente pequeño donde el barman sería el dueño,
sin embargo, la aplicación puede evolucionar fácilmente, añadiendole secciones
más ligadas a caja y pedidos pagados o incluir un menú de ayuda con tragos (fotos)
y precios, que sirva de apoyo para los meseros novatos.
